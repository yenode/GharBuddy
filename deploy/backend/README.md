# GharBuddy Backend Deployment

## Quick Deploy to Render

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy)

### Manual Setup

1. **Fork or connect** `yenode/GharBuddy` to Render
2. **New Web Service** → connect repo
3. **Build Command:** `pip install -r Backend/requirements.txt`
4. **Start Command:** `python -m uvicorn Backend.MainFastApi:app --host 0.0.0.0 --port $PORT`
5. **Add env vars** from `render.yaml`

### Environment Variables Required

| Variable | Description |
|---|---|
| `MOCK_MODE` | `False` for production, `True` for demo |
| `DB_HOST` | Neon or RDS hostname |
| `DB_USER` / `DB_PASSWORD` / `DB_NAME` | PostgreSQL credentials |
| `AWS_ACCESS_KEY_ID` / `AWS_SECRET_ACCESS_KEY` | IAM credentials for Bedrock |
| `AWS_DEFAULT_REGION` | `us-east-1` |
| `JWT_SECRET` | Random 32-char secret (auto-generated by Render) |

### AWS App Runner Alternative

```bash
# Build and push Docker image
docker build -t gharbuddy-backend .
aws ecr create-repository --repository-name gharbuddy-backend
docker tag gharbuddy-backend:latest <account>.dkr.ecr.us-east-1.amazonaws.com/gharbuddy-backend:latest
docker push <account>.dkr.ecr.us-east-1.amazonaws.com/gharbuddy-backend:latest

# Create App Runner service via AWS Console or CLI
aws apprunner create-service \
  --service-name gharbuddy-api \
  --source-configuration '{"ImageRepository":{"ImageIdentifier":"<ecr-uri>:latest","ImageConfiguration":{"Port":"8000"},"ImageRepositoryType":"ECR"}}'
```
