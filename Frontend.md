# Frontend Client Application:

The front-end client is built using React, Vite, and vanilla CSS styling. It provides a visual control console for managing home automation, inspecting AI logic, tracking microgrid metrics, and logging in securely.

## Page Descriptions:

The application contains five main pages wrapped in a common sidebar layout:

### 1. Landing Gateway Page:
- Accessible at `/`.
- Acts as a landing page explaining core functionalities.
- Connects users directly to the login interface.

### 2. Secure Login Page:
- Accessible at `/login`.
- Standard username and password inputs validated on submit.
- Integrated Google Login button utilizing `@react-oauth/google` context wrapper.
- Demo bypass route for easy presentation evaluation.

### 3. Digital Twin Overview:
- Accessible at `/app`.
- Hosts an interactive 2D floor plan of an Indian home (prayer room, kitchen, bathrooms, hallways).
- Displays animated indicators for device toggles (on or off) and resident occupancy.
- Shows dynamic sensor logs for flushes, motion sensors, and temperature updates.

### 4. Smart Appliance Manager:
- Accessible at `/app/devices`.
- Lists connected devices including geysers, ACs, water pumps, pooja lights, and speakers.
- Offers instant sliders, toggles, and usage statistics.
- Houses quick-preset activities like pooja prayer setup and kids study sessions.

### 5. Energy and Outages Analytics:
- Accessible at `/app/energy`.
- Visualizes inverter charge percentages and real-time solar generation logs.
- Displays predictive 30-minute load-shedding warning widgets.
- Outlines financial savings and peak load reductions in rupees.

### 6. AI Agent Reasoning Logs:
- Accessible at `/app/insights`.
- Outputs active thinking lines generated by Bedrock and Gemini reasoning routines.
- Includes a RAG similarity visualizer detailing which rules matched the current triggers.
- Offers direct switches to trigger simulated events (e.g. toilet flushes, cooker whistles).
- Houses caregiver status resets and telemetry configurations.

### 7. Community Grid Sharing:
- Accessible at `/app/community`.
- Renders local microgrid telemetry representing neighboring houses.
- Displays interactive charts for solar exports, imports, and local grid demands.

## Data State Providers:

The application handles central data state synchronization:
- Data Context: Managed inside `DataContext.jsx` to distribute variables across pages.
- WebSocket Sync: Opens connection to `/api/ws/telemetry` for instant device changes.
- HTTP Polling Fallback: Defaults to polling every 5 seconds if WebSocket drops out.
- Token Storage: Persists authenticated JWT credentials in session storage.
